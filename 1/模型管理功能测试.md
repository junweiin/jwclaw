# 模型管理功能测试指南

## 新增功能

### 1. 编辑模型 ✏️
- 点击模型选择器（顶部栏的模型名称按钮）
- 在模型列表中找到要编辑的模型
- 点击模型名称右侧的 ✏️ 图标
- 修改模型信息（名称、API地址、密钥、描述）
- 点击"保存"按钮

**注意事项：**
- 如果修改了模型名称，系统会自动更新相关配置
- 如果编辑的是当前正在使用的模型，config.json也会同步更新
- 新名称不能与其他已有模型重复

### 2. 删除模型 🗑️
- 点击模型选择器
- 在模型列表中找到要删除的模型
- 点击模型名称右侧的 🗑️ 图标
- 确认删除操作

**限制条件：**
- ❌ **不能删除当前正在使用的模型**
- 如需删除当前模型，请先切换到其他模型

### 3. 切换模型
- 每个模型项底部都有独立的"切换"按钮
- 当前使用的模型显示"✓ 当前使用"
- 其他模型显示"切换到该模型"
- 点击后需要重启JwClaw才能生效

## 测试场景

### 场景1：编辑非当前模型
1. 打开模型选择器
2. 点击非当前模型的 ✏️ 按钮
3. 修改API地址或描述
4. 保存并验证更新成功

### 场景2：编辑当前模型并重命名
1. 打开模型选择器
2. 点击当前模型的 ✏️ 按钮
3. 修改模型名称
4. 保存后检查：
   - models.json中的current字段已更新
   - config.json中的model字段已更新
   - 顶部栏显示的模型名称已更新

### 场景3：尝试删除当前模型
1. 确保某个模型是当前使用的
2. 尝试删除该模型
3. 应该看到错误提示："不能删除当前正在使用的模型，请先切换到其他模型"

### 场景4：删除非当前模型
1. 切换到模型A
2. 删除模型B
3. 验证：
   - 模型B从列表中消失
   - models.json中不再包含模型B
   - 当前仍使用模型A

### 场景5：添加新模型
1. 点击"+ 添加新模型"
2. 填写所有必填字段
3. 保存并验证新模型出现在列表中

## API端点

### POST /api/model/edit
**请求体：**
```json
{
  "original_name": "原模型名称",
  "name": "新模型名称",
  "api_base": "https://api.example.com/v1",
  "api_key": "your-api-key",
  "description": "模型描述"
}
```

**响应：**
```json
{
  "success": true,
  "message": "已更新模型: 新模型名称"
}
```

### POST /api/model/delete
**请求体：**
```json
{
  "model_name": "要删除的模型名称"
}
```

**响应（成功）：**
```json
{
  "success": true,
  "message": "已删除模型: 模型名称"
}
```

**响应（失败 - 当前模型）：**
```json
{
  "success": false,
  "error": "不能删除当前正在使用的模型，请先切换到其他模型"
}
```

## 常见问题

### Q: 为什么删除模型时提示不能删除？
A: 您试图删除的是当前正在使用的模型。请先切换到其他模型，然后再删除。

### Q: 编辑模型后，JwClaw需要使用新配置怎么办？
A: 如果只是修改API地址或密钥，重启JwClaw即可。如果修改了模型名称，系统会自动更新配置文件。

### Q: 编辑表单和添加表单有什么区别？
A: 
- 添加表单：用于创建全新模型，所有字段都是空的
- 编辑表单：用于修改现有模型，会预填充当前模型的信息

### Q: 如果models.json不存在会怎样？
A: 系统会从config.json自动创建初始的models.json文件，包含当前配置的模型。

## 技术实现

### 前端
- **editModel(modelName)** - 打开编辑表单并填充数据
- **saveEditedModel()** - 保存编辑后的模型
- **deleteModel(modelName)** - 删除模型（带确认对话框）
- **hideEditModelForm()** - 隐藏编辑表单

### 后端
- **/api/model/edit** - 处理模型编辑请求
- **/api/model/delete** - 处理模型删除请求
- 自动处理模型重命名时的配置同步
- 防止删除当前使用的模型

### 数据流
```
用户操作 → JavaScript函数 → Fetch API → Flask路由 → JSON文件更新 → 刷新列表
```
